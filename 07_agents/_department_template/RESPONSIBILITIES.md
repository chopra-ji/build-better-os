## Responsibility Changes

When a responsibility is added to or removed from this department, the following steps are required:

1. Update this file with the new responsibility or remove the old one.
2. Update `INPUTS.md` and `OUTPUTS.md` if the change affects the department's interface contract.
3. Update `WORKFLOW.md` if the change affects the operating flow.
4. Increment the department version in `VERSION.md` (MINOR increment for additions/removals).
5. Add an entry to `CHANGELOG.md`.
6. Notify all affected downstream departments of the change.
