# Examples

Annotated examples of high-quality entries in this domain.

## Purpose

Examples show what a completed, active-status entry looks like in this domain. They serve two purposes:

1. **Onboarding:** New contributors can read an example before writing their first entry to understand the expected depth, format, and reasoning quality
2. **Quality calibration:** When in doubt about whether an entry meets the standard, compare it to an example

## What Belongs Here

- Completed entries that exemplify the quality standard described in this domain's README
- Annotated versions of entries that explain *why* specific choices were made (useful for training new contributors)

## What Does NOT Belong Here

- Draft entries
- Entries for any purpose other than demonstrating quality standards

## How to Use Examples

**Before writing:** Read the example that corresponds to the entry type you are writing. Notice the level of specificity, the structure of the reasoning, and how the "Application at Build Better" section is handled.

**After writing:** Compare your draft against the example. If your entry is significantly thinner, ask yourself which sections need more depth before promoting to `status: active`.
