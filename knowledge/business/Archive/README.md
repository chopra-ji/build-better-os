# Archive

Entries that are deprecated, superseded, or no longer current.

## Purpose

The Archive holds entries that were once active but have been retired. Entries are archived — never deleted — because:

- Historical context is valuable: knowing what we once believed and why we changed is part of the knowledge record
- Archived entries may become relevant again if conditions change
- The archive prevents the same research from being repeated

## What Gets Archived

- Entries whose information is materially outdated and cannot be updated
- Entries superseded by a newer, better entry on the same topic
- Entries for tools, platforms, or methods we have stopped using
- Travel entries that have not been verified within 24 months (travel domain only)

## What Does NOT Get Archived

- Case studies — historical record, always retained in active form
- Campaign analyses — permanent record, never archived
- Techniques and principles — deprecate only if definitively disproven

## Archive Process

1. Add the following to the entry's frontmatter: `status: archived`
2. Add an archive note at the top of the entry explaining *why* it was archived and *when*
3. Move the file to this Archive/ directory
4. Remove the entry from INDEX.md's active tables
5. Add the entry to an "Archived" section at the bottom of INDEX.md

## Archive Note Format

```
> **Archived [YYYY-MM-DD]:** [Reason for archiving. Link to superseding entry if applicable.]
```
