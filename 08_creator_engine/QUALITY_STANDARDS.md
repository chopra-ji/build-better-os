# Quality Standards — Creator Engine

---

## Definition of Done

A script is approved for production when all of the following are true:

1. All eight pipeline stages have been completed
2. All eight dimension scores are ≥ 7
3. Aggregate score is ≥ 56 (70% of 80)
4. No dimension scored ≤ 4 at any point in the run
5. Estimated runtime is 45–60 seconds at 130 wpm
6. The Final Production Script contains no unresolved stage notes or placeholders
7. The change log accurately records every modification made during the run

A script that meets all criteria above is approved. A script that does not meet any single criterion is not approved, regardless of all other scores.

---

## Final Production Gate

Before the Final Production Script is released, the engine performs one final check:

| Check | Requirement |
|---|---|
| Runtime | 45–60 seconds (word count ÷ 130 wpm) |
| Hook | Present in opening sentence |
| Comment trigger | Identifiable and specific |
| CTA | Present in final 5 seconds |
| Aggregate score | ≥ 56 |
| Single-dimension minimum | All ≥ 7 |
| Change log | Complete — every modification recorded |
| Escalation | None open |

If any check fails, the script does not exit the pipeline. The failing check is logged and the appropriate stage re-run or escalation triggered.

---

## Production Notes

The Final Production Script is accompanied by production notes for the creator. These notes are brief — the script itself should be self-explanatory. Production notes cover:

- Any optimizations that require creator judgment to execute well (e.g., "The Stage 3 revision added a personal anecdote — confirm the specific figures are accurate before recording")
- Emphasis or pacing notes if the scoring surfaced rhythm concerns
- Caption copy, finalized and ready to paste
- On-screen text recommendations from Stage 8

Production notes are observations, not instructions. The creator makes the final call on delivery.

---

## What This Engine Does Not Guarantee

The Creator Engine guarantees script quality against defined criteria. It does not guarantee:

- **Performance.** A script that passes all eight stages is optimized for engagement — it is not guaranteed to perform. Audience response depends on factors outside the engine's control (account stage, timing, distribution, recording quality).
- **Uniqueness.** The engine reviews against Build Better's standards, not against every piece of content on Instagram.
- **Creator delivery.** A great script can be undermined by delivery. The engine's responsibility ends at the written script.

These limitations are not disclaimers. They are boundaries. The Creator Engine owns what it can control. It does not overclaim.
